# 찬양 플레이 리스트 통계 시각화 기능 기획 (곡 기준)

## 1. 목표 및 배경

- **목표**: 찬양 플레이 리스트 폴더(`playlist_path`) 안의 모든 `.pls` 리스트를 스캔해, **곡 단위** 통계를 수집·시각화한다. 리스트 기준 통계는 다루지 않는다.
- **배경**: 여러 리스트에 걸쳐 어떤 곡이 자주 쓰이는지, Key 분포는 어떤지 곡 관점에서 파악하기 어려우므로, **곡 기준** 통계·차트로 요약해 제공한다.

## 2. 데이터 소스 정리

| 소스 | 설명 |
|------|------|
| **플레이 리스트 폴더** | `self.playlist_path` — 여기 안의 모든 `.pls` 파일 (하위 폴더 포함 여부는 옵션) |
| **.pls 파일 구조** | JSON 배열. 항목은 `"상대경로"` 또는 `{ "path": "상대경로", "is_intermission": bool }` |
| **악보 루트** | `self.sheet_music_path` — `.pls`의 `path`는 이 루트 기준 상대경로 |
| **곡 메타데이터** | `song_metadata` DB / `metadata_cache`: `(song_key, lyrics)` 등 (선택 활용) |

**곡 단위 정의**
- 각 entry의 `path`를 `sheet_music_path`와 합쳐 **절대경로**로 만든다.
- **동일 절대경로 = 동일 곡**. 곡명은 파일명(base name)으로 표시.
- 통계는 모두 **곡(절대경로/곡명)** 단위로 집계한다.

## 3. 곡 기준 통계 항목

### 3.1 곡별 등장
- **곡별 등장 횟수**: 해당 곡이 **몇 개 리스트**에 포함되어 있는지
- **자주 쓰이는 곡 Top N**: 등장 횟수(리스트 개수) 순 정렬
- **한 리스트에만 등장하는 곡** vs **여러 리스트에 등장하는 곡** 구분 (또는 구간별 분포)

### 3.2 곡 요약 숫자
- **총 곡 수(중복 제거)**: 리스트 전체를 합쳤을 때 서로 다른 곡 개수
- **총 등장 횟수(중복 포함)**: 모든 리스트에서 곡이 나온 횟수 합
- **깨진 경로 수**: `path`가 실제 파일이 아닌 항목 수 (곡으로 집계 시 “존재하지 않는 곡” 수)

### 3.3 메타데이터 연동 (선택)
- **Key 분포**: 곡별 `song_key`가 있으면, Key별 **곡 수** (또는 등장 횟수 합) 시각화
- 가사/메모는 통계에는 사용하지 않고, 필요 시 툴팁 등으로만 노출

### 3.4 인터미션·품질 (선택)
- **인터미션 포함/제외 옵션**: 통계 집계 시 인터미션 항목을 포함할지 제외할지 선택
- **인터미션만 있는 곡** vs **일반 곡** 구분 표시 (필요 시)

## 4. 시각화 제안 (곡 기준)

### 4.1 필수 시각화
1. **곡별 등장 횟수 (리스트에 포함된 횟수)**  
   - 막대 그래프: X = 곡명(또는 상위 N개만), Y = 포함된 리스트 개수  
   - 예: “가장 많이 쓰인 곡 Top 20” ~ Top 50

2. **요약 숫자 카드 (곡 기준)**  
   - 총 곡 수(중복 제거), 총 등장 횟수(중복 포함), 깨진 경로(존재하지 않는 곡) 수

3. **곡 목록 테이블**  
   - 컬럼: 곡명, 등장 횟수(리스트 개수), (선택) Key  
   - 정렬: 등장 횟수 내림차순 등  
   - 곡 클릭 시: 해당 곡이 들어 있는 **리스트 목록** 표시

### 4.2 선택 시각화
4. **Key 분포** (메타데이터 있을 때)  
   - 파이 차트 또는 막대: Key별 **곡 수** (또는 등장 횟수 합)

5. **등장 횟수 구간 분포**  
   - 예: 1회만 등장한 곡 N개, 2~5회 N개, 6회 이상 N개 (막대 또는 파이)

## 5. UI/UX 제안 (곡 중심)

### 5.1 진입점
- **메뉴**: `통계` → `플레이 리스트 곡 통계 보기`
- **플레이 리스트 패널**: 우클릭 메뉴에 `플레이 리스트 곡 통계 보기`
- 동일 **통계 다이얼로그(또는 서브 윈도우)** 로 진입

### 5.2 통계 화면 구성 (곡 기준만)
- **요약**: 곡 기준 숫자 카드 + (선택) Key 분포 차트
- **곡별 차트**: 곡별 등장 횟수 막대 그래프 (Top N 조절 가능)
- **곡 목록 테이블**: 곡명, 등장 횟수, Key(선택) + 클릭 시 “이 곡이 포함된 리스트” 목록
- **필터/옵션**  
  - 인터미션 포함/제외  
  - Top N, 정렬 기준 (등장 횟수/곡명)  
  - 하위 폴더 `.pls` 포함 여부

### 5.3 기술 스택 제안
- **차트**: PySide6 `QChart`(Qt Charts) 또는 matplotlib 임베드
- **표**: `QTableWidget` 또는 `QTableView` + 모델
- **비동기**: `.pls`가 많으면 스캔을 `QThread` 등으로 분할해 UI 멈춤 방지

## 6. 데이터 수집 흐름 (곡 기준)

1. `playlist_path`에서 `.pls` 목록 수집 (재귀/비재귀 옵션).
2. 각 `.pls`를 순회하며:
   - JSON 로드, entry 순회.
   - 각 entry에서 `path`, `is_intermission` 추출 (인터미션 제외 옵션 적용).
   - `full_path = os.path.normpath(os.path.join(sheet_music_path, path))` → **곡 식별자**.
   - 전역 집계:
     - **곡 → 등장 리스트 이름 목록** (또는 등장 횟수만) 갱신.
     - 파일 존재 여부로 “깨진 경로” 집계.
3. (선택) `metadata_cache` 또는 DB에서 `full_path`로 Key 조회 → Key별 곡 수/등장 횟수 집계.
4. 집계 결과를 **곡 단위**로 정리해 차트·테이블에 전달.

## 7. 단계별 구현 제안 (곡 기준)

| 단계 | 내용 |
|------|------|
| 1 | 데이터 수집: `playlist_path` 스캔 → **곡별 등장 리스트/등장 횟수** 집계, 깨진 경로 수 |
| 2 | 요약 숫자 카드(곡 수, 총 등장 횟수, 깨진 경로) + **곡별 등장 횟수** 막대 그래프 |
| 3 | 곡 목록 테이블 + 곡 클릭 시 “포함된 리스트” 목록, 인터미션 포함/제외 옵션 |
| 4 | (선택) Key 분포 차트, 등장 횟수 구간 분포 |
| 5 | (선택) CSV/이미지 내보내기 |
